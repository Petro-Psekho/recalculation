# Corrugated Box Internal → External Size Calculator

Веб-приложение на чистом **HTML + CSS + JavaScript** для ориентировочного пересчета внутренних размеров гофроящика в расчетный наружный размер, safe size for Cape Pack и производственный максимум.

Инструмент рассчитан на технологов упаковки, packaging development managers, препресс-специалистов и всех, кто работает с гофроящиками, шоу-боксами, лотками и транспортной гофротарой.

## Возможности

- Пересчет внутреннего размера ящика в расчетный наружный размер.
- Учет профиля гофрокартона и толщины картона.
- Предустановленные профили: `E`, `B`, `C`, `EB`, `BC`, `Custom`.
- Предустановленные марки картона: `T-21`, `T-22`, `T-23`, `T-24`, `P-31`, `P-32`, `Custom`.
- Автоподстановка рекомендуемого производственного допуска по марке картона.
- Предустановленные конструкции: `FEFCO 0201`, `FEFCO 0427`, `FEFCO 0401`, `Custom`.
- Ручная корректировка количества стенок по длине, ширине и высоте.
- Расчет safe size for Cape Pack с логистическим запасом.
- Расчет производственного максимума.
- Расчет максимально безопасного размера.
- Проверка паллетизации для паллеты, например `1200 × 800 мм`.
- Расчет лучшей ориентации ящиков на паллете.
- Предупреждения о риске нависания и схемах, которые стоят почти “в ноль”.
- Простая визуализация слоя паллеты.
- Копирование результата в буфер обмена.
- Экспорт результата в JSON.
- Адаптивная верстка для desktop и mobile.

## Формулы

### Расчетный наружный размер

```js
outerLength = innerLength + thickness * lengthWalls;
outerWidth = innerWidth + thickness * widthWalls;
outerHeight = innerHeight + thickness * heightWalls;
```

### Safe size for Cape Pack

```js
capeLength = outerLength + logisticsAllowance;
capeWidth = outerWidth + logisticsAllowance;
capeHeight = outerHeight + logisticsAllowance;
```

### Производственный максимум

```js
maxLength = outerLength + tolerance;
maxWidth = outerWidth + tolerance;
maxHeight = outerHeight + tolerance;
```

### Максимально безопасный размер

```js
safeLength = outerLength + tolerance + logisticsAllowance;
safeWidth = outerWidth + tolerance + logisticsAllowance;
safeHeight = outerHeight + tolerance + logisticsAllowance;
```

## Паллетизация

Приложение считает стандартную и повернутую ориентацию ящика на паллете:

```js
casesAlongLength = Math.floor(palletLength / caseLength);
casesAlongWidth = Math.floor(palletWidth / caseWidth);
casesPerLayer = casesAlongLength * casesAlongWidth;

layers = Math.floor((maxPalletHeight - palletHeight) / caseHeight);
totalCases = casesPerLayer * layers;
```

Также выводятся:

- ящиков в слое;
- количество слоев;
- всего ящиков на паллете;
- лучшая ориентация;
- остаток свободного места;
- предупреждения по рискам паллетизации.

## Тестовый пример по умолчанию

При загрузке страницы подставляется пример:

| Параметр | Значение |
|---|---:|
| Внутренняя длина | `595 мм` |
| Внутренняя ширина | `260 мм` |
| Внутренняя высота | `150 мм` |
| Профиль гофрокартона | `B-flute` |
| Толщина | `3 мм` |
| Марка картона | `T-22` |
| Конструкция | `FEFCO 0201` |
| Производственный допуск | `3 мм` |
| Логистический запас | `2 мм` |
| Паллета | `1200 × 800 мм` |

Ожидаемый ориентировочный результат:

- расчетный наружный размер: примерно `601 × 266 × 156 мм`;
- safe size for Cape Pack: примерно `603 × 268 × 158 мм`;
- схема `2 × 3` может быть рискованной по ширине, потому что `268 × 3 = 804 мм`.

## Как запустить локально

Backend не нужен.

1. Скачайте или клонируйте репозиторий.
2. Откройте файл `index.html` в браузере.

Можно также запустить простой локальный сервер:

```bash
python -m http.server 8000
```

После этого открыть:

```text
http://localhost:8000
```

## Важное предупреждение

Расчет является инженерной оценкой и не должен называться абсолютно точным. Финальные наружные размеры нужно подтверждать замером готового закрытого ящика из производственной партии.

Для Cape Pack рекомендуется использовать максимальный фактический размер + логистический запас `2–3 мм`.

Количество стенок по длине, ширине и высоте зависит от конструкции и реальной технологии изготовления. Эти значения нужно проверять по чертежу, развертке или спецификации поставщика.
